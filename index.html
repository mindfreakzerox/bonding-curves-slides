Bonding Curves & Dynamic Token Pricing

Based on speedrunethereum guide + Web3 Solidity patterns

Solidity Tokenomics DeFi

Why Dynamic Pricing?

  • Fixed price launches: gas wars, unfair distribution, poor price discovery
  • Dynamic pricing reacts to demand/supply in-contract
  • Always-on liquidity: contract buys/sells at curve price
  • Transparent & predictable: math is public

What is a Bonding Curve?

  • Function P(S) sets price based on current supply S
  • Buy: mint → supply ↑ → price rises
  • Sell: burn → supply ↓ → price falls
  • Contract is the counterparty; no order book needed

Common Curve Shapes

Linear

P = mS + b

  • Steady, predictable
  • Community tokens

Exponential

P = a · e^{kS}

  • Cheap early, ramps fast
  • Hype/early adopter heavy

Logarithmic

P = k · ln(S + c)

  • Rises fast, then stabilizes
  • Bootstraps then levels

Others

  • Polynomial (quadratic)
  • Sigmoid (slow–fast–plateau)
  • Step/Hybrid

Use Cases

  • Token launches (ICO/IDO), NFT mints
  • Community/DAO tokens
  • AMM-like continuous liquidity
  • RWA tokenization, DeSci funding
  • Continuous funding / social bonds

Design Levers

  • Slope/shape (linear vs expo vs log)
  • Base price / offset (b or c)
  • Fees/treasury cut on buys/sells
  • Max supply caps & reserve ratios
  • Buy/sell spread to discourage griefing

Solidity Building Blocks

  • Track total supply (S) and reserves (ETH/erc20)
  • Pricing function P(S) or integral for cost
  • Buy: compute cost to move from S to S+Δ
  • Sell: compute refund from S to S-Δ
  • Events: Bought, Sold (buyer, amount, cost, price)
  • Security: reentrancy guard, overflow-safe math

Example: Linear Curve (Pseudo-Solidity)

function price(uint256 s) public view returns (uint256) {
    return m * s + b; // m, b set at deploy
}

function costToMint(uint256 amount) public view returns (uint256) {
    // integral of price from S to S+amount
    uint256 s1 = totalSupply;
    uint256 s2 = totalSupply + amount;
    return m * (s1 + s2 + 1) * amount / 2 + b * amount;
}

Integral approach avoids loops; adapt for expo/log curves using approximations.

Practical Tips

  • Use fixed-point math (e.g., 1e18 scale) for exponentials/logs
  • Cap max mint per tx to avoid overflow/DoS
  • Charge protocol fee → treasury
  • Consider circuit breakers / pausable
  • Unit test edge cases: start/end supply, large buys, sells

Risks & Gotchas

  • Front-running & sandwiching around large buys/sells
  • Price manipulation if reserve is small
  • Precision loss in exponentials/logs
  • Liquidity drain if sell path not fee-adjusted
  • Economic griefing: zero-liquidity start, sudden exits

Demo Structure (suggested)

  • Start with linear curve mint/burn functions
  • Show price progression as supply grows
  • Add fee + treasury collection
  • Discuss switch to exponential/log variant

Key Formulas (Linear)

  • Price: P(S) = mS + b
  • Buy cost from S → S+Δ: ∫ P(s) ds = m/2 · ( (S+Δ)^2 - S^2 ) + bΔ
  • Sell refund symmetric; apply fee/discount
